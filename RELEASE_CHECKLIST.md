# 智慧教育智能体平台发行加固清单

当前版本已按“校内可演示 / 小范围内测”方向补齐第一批发行基线，但仍不是完整商业 SaaS 发行版。正式面向多学校、多班级长期运行前，应继续按本清单推进。

## 已落地的发行基线

- 真实登录态：后端签发 `HttpOnly` session cookie，`/api/state` 根据会话取当前用户。
- 密码安全：账号密码使用 Node 内置 `scrypt` 哈希存储，旧明文密码会在启动/登录时自动迁移。
- 基础 RBAC：支持 `admin`、`teacher`、`student` 三类角色；普通用户不能伪造 `userId`、`teacherId`、`studentId` 操作别人数据。
- 登录保护：失败次数达到阈值后进入冷却期。
- 审计日志：登录、上传、图谱生成/删除、资料入库/删除、对话、错题、掌握度、聊天、班级、作业、批改等敏感操作写入 `auditLogs` 和 `logs/audit.log`。
- 安全响应头：API 和静态页面增加基础安全头、CSP、点击劫持防护和 MIME sniffing 防护。
- 健康检查：新增 `/api/healthz`、`/api/readyz`。
- 原子写库：`data/db.json` 写入改为临时文件 + rename，降低写坏文件概率。
- 上传保护：上传入口增加后端文件格式白名单和基础文件头校验，阻断可执行文件、脚本类文件和伪装格式。
- 部署入口：新增 `.env` 自动加载、Dockerfile、Docker Compose 和 `DEPLOYMENT.md`。
- 最小烟测：`npm test` 会启动测试服务，验证健康检查、登录态、越权阻断、恶意上传拦截和 AI 对话结构。

## 仍需继续推进的 P0 项

- 数据库迁移：将 `data/db.json` 迁移到 SQLite WAL 或 PostgreSQL，并建立 migration。
- 文件与任务持久化：上传会话、OCR、图谱生成任务应进入数据库/任务队列，避免服务重启丢任务。
- 真实 RAG：引入 embedding、混合检索、rerank 和真实 PDF 页码映射。
- 真实 LLM：接入可配置模型供应商，增加流式输出、停止生成、token 预算和模型降级。
- 文件安全：当前已有基础文件头校验；后续仍需补病毒扫描、压缩炸弹防护、页数/时间/容量配额和租户级容量限制。
- 作业批改：增加 rubric、多题评分、教师最终确认、申诉/复核和成绩导出。
- 工程化前端：将单文件 SPA 拆为 Vite + 组件化 + TypeScript + Playwright。

## 小范围内测建议

1. 使用 `.env` 配置强随机 `SESSION_SECRET`，不要使用默认开发密钥。
2. 通过反向代理启用 HTTPS，并设置 `COOKIE_SECURE=true`。
3. 定期备份 `data/` 和 `logs/audit.log`。
4. 上传资料只使用有授权的教材、课件、讲义。
5. 以教师确认结果作为正式作业分数，AI 批改仅作为建议。
6. 每次发布前运行：

```bash
npm run check
npm test
```

## 发行路线

- 第 1 阶段：校内可演示版。重点是权限、密码、部署、审计和健康检查。
- 第 2 阶段：小范围内测版。重点是数据库、任务队列、真实 RAG、上传恢复。
- 第 3 阶段：教师可用版。重点是作业闭环、班级统计、教师确认批改。
- 第 4 阶段：平台发行版。重点是多租户、监控、备份、合规、自动化测试和文档。
